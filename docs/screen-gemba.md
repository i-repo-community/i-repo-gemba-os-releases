# 現場蓄積データ

送信が終わったデータを、アプリ内で**読み取り専用**で閲覧・検索する画面です。AI が読むのと同じ仕組みを、人間が画面で見られるようにしたものです。

- データセットを選ぶと一覧が表示されます。
- **詳細** ボタンで、項目（クラスター）の値・添付ファイル・元データを確認できます。
- 送り先のパスやパスワードなどの接続情報は表示しません（安全のため）。

## 実行履歴（旧・実行状況）

蓄積先カードの下に、その配信先の**実行ログ・成功確認・再実行**が常設されています。カードを押すと、その配信先に
絞った実行履歴が見られます（もう一度押す・× で全件表示に戻ります）。共有エクスポートはページ最下部にあります。

> **「成功」の見分け方**：このアプリでは、**送り先が「確かに入りました」と返した確認（レシート）が取れたときだけ
> 成功**とみなします。処理が途中で終わってもエラーにならない場合があるため、この実行履歴の成功表示で判断して
> ください。

## チームへの共有

配信済みデータを、他のメンバーへ**読み取り専用**で渡せます。ページ最下部の共有欄で操作します（管理者向け）。

- **自動共有**：トグルで有効にすると、配信が成功するたびに共有データが自動で更新されます（最終共有の時刻・
  世代を表示）。オフの間は「今すぐ共有」を押したときだけ更新されます。
- **＋チケットを発行**：宛先・読める範囲（データセット）・有効期限（30日／90日／1年）を選ぶと、チケットが
  発行されコピーできます。全文が見えるのはその場限りで、発行台帳には残りません。渡す相手（メンバー）は、
  Slack などで受け取ったチケットを共有ビューに貼るだけで読めるようになります。
  - GCS の配信先で発行するには、読み手用のサービスアカウント（メール）を入力します（次回のため記憶されます）。
    発行に使う接続の資格には、その SA 限定の鍵管理権限（`roles/iam.serviceAccountKeyAdmin`）を付与しておく
    必要があります。
  - S3／Azure の配信先は、あらかじめ接続の設定に読み取り専用の資格（reader access key／SAS）を追加してから
    発行します。
- **発行済みチケット（発行台帳）**：誰に・どの範囲を・いつまで渡したかを一覧でき、1枚ずつ**失効**できます
  （保存先の種類によっては個別に失効できず、期限切れを待つ必要があります）。
