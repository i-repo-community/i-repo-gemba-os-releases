# 現場蓄積データ

送信が終わったデータを、アプリ内で**読み取り専用**で閲覧・検索する画面です。AI が読むのと同じ仕組みを、人間が画面で見られるようにしたものです。v0.30.0 で「配る」機能（共有エクスポート・チケット発行）は [API](screen-readapi.html) 画面へ移り、この画面は**読むこと専用**になりました。

- **データソース**の切り替え：既定は「この機のデータ」（自テナントの配信済みデータ）です。他の機・他システムが公開した読み取り API をこの画面の「APIソース」区画から**ソースとして登録**すると、切り替えて検索できます。**すべてのソース**を選ぶと、登録済みの各ソースのデータセット一覧をソース名バッジ付きで1画面に合成表示します（1クエリでの横断検索ではなく在庫の一覧です。データセットを選ぶと、そのデータセットが属すソースへ切り替えて通常の検索に入ります）。
- データセットを選ぶと一覧が表示されます。
- **詳細** ボタンで、項目（クラスター）の値・添付ファイル・元データを確認できます。検索結果からは成果物（添付ファイルなど）のダウンロードもできます。
- 送り先のパスやパスワードなどの接続情報は表示しません（安全のため）。

> 蓄積先カードの下に、その配信先の**実行ログ・成功確認・再実行**は引き続き常設されています。**「成功」の見分け方**：このアプリでは、**送り先が「確かに入りました」と返した確認（レシート）が取れたときだけ成功**とみなします。処理が途中で終わってもエラーにならない場合があるため、この実行履歴の成功表示で判断してください。

> **チームへの共有**（共有エクスポート・チケット発行）は [API](screen-readapi.html) 画面へ移りました。設定の「使い方」欄で「従来の共有（snapshot 配布）を表示」を ON にしている間は、API 画面の下部に引き続き表示されます。
