# 初回セットアップ

## 事前に必要なもの

- **アプリ本体**: [ダウンロードページ](https://github.com/i-repo-community/i-repo-gemba-os-releases/releases/latest)のインストーラから入れます。
- **i-repo CLI**（＋実行環境の **Node.js**）: データ抽出に使う土台です。アプリには**同梱されない**ため、画面左下の `i-repo CLI` が緑にならない場合は導入してください（普段アプリだけ使う方も、入っている必要はあります。コマンドを手で打つ運用は不要です）。

  ```bash
  npm install -g i-repo
  ```

  詳細は [i-repo CLI のページ](i-repo-cli.html) を参照してください。

> 送り先プラグインはアプリに同梱されており、下の手順4の**ボタンで導入**します（別途ダウンロード不要）。

## セットアップ手順

1. アプリを起動します。
2. 初回は **設定** 画面が自動で開きます。
3. **i-Reporter の接続先（API エンドポイント）**・**ユーザーID**・**パスワード** を入力して保存します。
4. プラグイン（送り先ごとの部品）を導入します。**[プラグイン](screen-plugins.html)画面の「公式プラグインを導入」ボタンを押すだけ**です。
   プラグインはアプリに同梱されているので、ダウンロードやコマンドは不要です（新しい版が同梱されたときは同じ画面に「更新あり」が出ます）。
5. 画面左下の状態表示で **`i-repo CLI` が緑** になっていれば準備完了です。

> プラグインは「送り先の種類」ごとの部品です。使う送り先のプラグインだけ入っていれば動きます（[プラグイン](screen-plugins.html)画面で状態を確認できます）。

> 開発者向け（ソースを取得している場合）: プラグインは `plugins/install.ps1`（Windows）/ `plugins/install.sh`（macOS・Linux）でも導入できます。配布アプリ単体には install スクリプトは含まれないため、通常は上記のボタン導入を使ってください。
