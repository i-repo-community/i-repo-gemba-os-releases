# 初回セットアップ

## 事前に必要なもの

- **アプリ本体**: [ダウンロードページ](https://github.com/i-repo-community/i-repo-gemba-os-releases/releases/latest)のインストーラから入れます。
- **i-repo CLI**（＋実行環境の **Node.js**）: データ抽出に使う土台です。アプリには**同梱されない**ため、画面左下の `i-repo CLI` が緑にならない場合は導入してください（普段アプリだけ使う方も、入っている必要はあります。コマンドを手で打つ運用は不要です）。

  ```bash
  npm install -g i-repo
  ```

  詳細は [i-repo CLI のページ](i-repo-cli.html) を参照してください。

> 送り先プラグインはアプリに同梱されており、下の手順4の**ボタンで導入**します（別途ダウンロード不要）。

<a id="install-warning"></a>

## インストール時の警告について（署名対応までの間）

現在、アプリの**デジタル署名を準備中**です（Windows は署名の申請手続き中、macOS は署名・公証を準備中）。
そのため初回のインストール・起動時に、OS が「発行元を確認できない」旨の警告を出します。アプリ自体は
[公式のダウンロードページ](https://github.com/i-repo-community/i-repo-gemba-os-releases/releases/latest)（GitHub）でのみ配布しています。**次の手順で起動できます**。署名（macOS では公証も）が有効になれば、これらの警告は**原則として出なくなります**。

### Windows（SmartScreen）

1. ダウンロードしたインストーラ（`…-setup.exe` / 軽量版 `…-setup-online.exe`）を実行します。
2. **「WindowsによってPCが保護されました」**（Microsoft Defender SmartScreen）が表示されたら、
   青い文字の **「詳細情報」** をクリックします。
3. 下に出る **「実行」** ボタンを押すと、インストールが始まります。
4. インストール後、**アプリを初めて起動するとき**にも同じ警告が出ることがあります。その場合も同様に
   **「詳細情報」→「実行」** で起動できます。

> Microsoft Edge でダウンロード時に「一般的にダウンロードされていません」と出る場合は、ブラウザの
> ダウンロード一覧で該当ファイルの **「…」→「保持する」** を選んでから実行してください。

### macOS（Gatekeeper）

1. ダウンロードした `…universal.dmg` を開き、アプリを **「アプリケーション」** フォルダへドラッグします。
2. 初回だけ、アプリを **右クリック（Control＋クリック）→「開く」** を選び、確認ダイアログで **「開く」** を押します
   （ダブルクリックではなく右クリックから開くのがポイントです）。
3. それでも「開発元を確認できません」等で開けないときは、**アップルメニュー →「システム設定」→
   「プライバシーとセキュリティ」** を開き、画面下部の「"i-Repo GEMBA OS" は…」の横の **「このまま開く」**
   をクリックして、管理者パスワードで許可します。

> macOS のバージョンによって表示文言や場所が少し異なる場合があります。

## セットアップ手順

1. アプリを起動します。
2. 初回は **設定** 画面が自動で開きます。
3. **i-Reporter の接続先（API エンドポイント）**・**ユーザーID**・**パスワード** を入力して保存します。
4. プラグイン（送り先ごとの部品）を導入します。**[プラグイン](screen-plugins.html)画面の「公式プラグインを導入」ボタンを押すだけ**です。
   プラグインはアプリに同梱されているので、ダウンロードやコマンドは不要です（新しい版が同梱されたときは同じ画面に「更新あり」が出ます）。
5. 画面左下の状態表示で **`i-repo CLI` が緑** になっていれば準備完了です。

> プラグインは「送り先の種類」ごとの部品です。使う送り先のプラグインだけ入っていれば動きます（[プラグイン](screen-plugins.html)画面で状態を確認できます）。

> 開発者向け（ソースを取得している場合）: プラグインは `plugins/install.ps1`（Windows）/ `plugins/install.sh`（macOS・Linux）でも導入できます。配布アプリ単体には install スクリプトは含まれないため、通常は上記のボタン導入を使ってください。
