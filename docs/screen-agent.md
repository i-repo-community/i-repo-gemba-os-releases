# 現場 AI チャット

アプリの中で、**現場データについて AI に日本語で質問**できる画面です。お使いの AI（手元に導入した CLI）をそのまま使うため、推論の料金はお客様のアカウント側に発生します（このアプリが勝手に課金することはありません）。

- 「先月の点検で指摘が多かった項目は？」のように、自然な言葉で聞けます。
- **「月別の件数」「削除区分ごとの内訳」「点検種類の数」「合計・平均」などの集計**も、全行を取り出さず配信先（**SQLite / Elasticsearch / MongoDB / Parquet / BigQuery**）側で集計して答えます（大量でも軽く、消費トークンも少なく）。
- AI は[現場蓄積データ](screen-gemba.html)を**読み取り専用**で参照します。データを書き換えたり消したりはしません。
- 回答の下に、その質問で使った**トークン数の目安**を控えめに表示します（課金の参考に）。
- 操作に迷ったら、チャット下部の**「画面の使い方を聞く」**や、どの画面でも右上の**「操作をAIに聞く」**から、必要な設定の場所と順番を画面上で案内してもらえます。

## 現場スキルを「レンズ」として装着する

確定済みの**現場スキル**（[Ontology Studio](screen-ontology.html) で作った“ベテランの見方”）があるときは、入力欄の**左端**に **◎ 現場スキル** ボタンが出ます。押すとレンズのように並んだスキルから選べ、選ぶとそのままレンズとして**装着**されます（ボタンが、装着したスキル名の入ったチップに変わります。**✕** で外せます）。

- 装着すると、そのスキルの帳票たち（識別子と絞り条件）を持った状態で質問できます。**帳票をまたいだ質問候補**も自動で出ます。
- 装着しなくても大丈夫です。質問の内容に関係するスキルは、**必要なときだけ自動で読み込まれます**（会話は軽いまま）。
- スキルの中身（確認済みの見方）は必要時のみ参照され、**帳票の値や個人情報がAIへ送られることはありません**。
- Claude 専用ではなく、**Codex** でも同じスキルをそのまま使えます（i-repo CLI 2.30.0 以降が必要です）。

## 応答の速さ・使う AI を選ぶ（Claude / Codex）

入力欄の**右下のメニュー**から、**応答の速さ**（既定は「速い」・手間のかかる突き合わせなら「じっくり」）と、使う AI を **Claude** と **Codex** から切り替えられます（導入済みのものだけ選べます）。どちらの設定も保存され、次回も引き継がれます。

- **Claude**（Claude Code CLI）: 既定。`claude auth login` でサインインして使います。
- **Codex**（OpenAI Codex CLI）: `codex login` でサインインして使います。**試験的対応**です（下記の注意を参照）。

Claude Code が Bedrock / Vertex / Foundry などのクラウドプロバイダや Anthropic API を使うよう設定されている場合、現場 AI チャットの Claude はそのプロバイダをそのまま自動的に使います。認識状況は準備状況パネルに表示され、認証はそのクラウド側（AWS / GCP / Azure）で行われます（`claude auth login` ではありません）。詳しくは [AI 連携の詳細](manual-ai.html) を参照してください。

切り替えると会話は新しく始まります（それまでのやりとりは引き継ぎません）。

> **Codex を使うときの注意（試験的）**
> Codex は読み取り専用モードで起動しますが、Claude のように「アプリの読み取りツールだけに限定する」仕組みが CLI 側にないため、防御は Claude より緩めです（書き込み・破壊は抑止されます）。現場データへの読み取り質問用途でお使いください。安心して常用したい場合は Claude をおすすめします。

準備（導入・ログイン）が未完了のときは、画面上部に必要な手順（導入コマンド・`login` コマンド）が表示されます。選んだ AI に応じて案内が切り替わります。
