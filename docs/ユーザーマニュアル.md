# i-Repo GEMBA OS ユーザーマニュアル

**i-Repo GEMBA OS** は、i-Reporter で集めた現場の帳票データを、そのままにせず「使える形」にするためのデスクトップアプリです（Windows / macOS）。むずかしい設定やプログラミングは要りません。

できることは、大きく3つです。

| やりたいこと | このアプリでの言い方 | ざっくり言うと |
|---|---|---|
| 帳票データを社内のデータ置き場へ自動で送りたい | **配信（フロー）** | 「どの帳票を・いつ・どこへ」を決めておくと、自動で送り続けます |
| 送ったデータが本当に入ったか確認したい | **帳票ライブビュー / 現場蓄積データ** | 送信元と送信先を突き合わせ、中身もアプリ内で見られます |
| データについて人に聞くように質問したい | **現場 AI チャット** | 「先月の不具合報告を要約して」「月別の件数は？」のように、要約も**集計**も AI に日本語で聞けます |

送り先（データ置き場）は、お客様の環境に合わせて選べます — **SQLite / Parquet / MongoDB / Elasticsearch / BigQuery / Amazon S3 / Google Cloud Storage / Azure Blob / ファイル共有** など。配信済みを確認した古い帳票は、削除プラグインで安全に整理できます。

## 表示バッジの意味と対処

画面に出る小さなバッジ（しるし）の意味と、消し方です。どちらも**実行には支障ありません**。

<a id="version-changed"></a>

### 版変更バッジ

フロー作成画面で、抽出に使うプラグインが**フロー作成時の版から新しい版に更新された**ときに出るお知らせです（例：`版変更 v1.2→v1.3`）。**配信はそのまま正常に動きます。**

- 消し方：そのフローを**開いて保存し直す**と、記録している版が現行に更新されて消えます。バッジ横の「**解消**」ボタンでも同じことができます（配信内容は変わりません）。
- 再実行では消えません（実行は記録版ではなく現行版で動くため、表示だけのお知らせです）。

<a id="unknown-provenance"></a>

### 出所不明 / 紐付け

「帳票ライブビュー」で、**接続先（i-Reporter サーバー）の識別が無い古い配信実績**に付くしるしです。接続先を切り替えた後などに、どのサーバー由来か判断できない記録を、誤って別サーバーの記録として扱わないために区別しています。

- **「未書き出し（実績なし）」との違い**：出所不明は「**配信実績はあるが出所が不明**」、未書き出しは「**そもそも配信実績が無い**」です。
- 対処：今の接続先のものだと分かっていれば、右の「**紐付け**」で現在の接続先の記録として確定できます。確定後はこの画面でのソフトデリート対象に含められます（別サーバー由来の記録は変更されません）。
