# i-repo-upload 使い方

> PDF / Excel を、利用者がそのまま開ける名前でファイル共有先へ配置する（macOS・Windows・Linux 対応）

archive が作った保管フォルダの `documents/` だけを取り出し、PDF / Excel を送り先へ**実名のまま**配置するプラグインです。
管理用の manifest や receipt は置かないため、Box・Google Drive・SharePoint などの同期フォルダや、
S3 / GCS / Azure Blob を人がファイル置き場として使う場合に向いています。

## archive との違い

| | archive | upload |
|---|---|---|
| 主な目的 | 帳票データ一式を証跡として保管 | PDF / Excel を人が開きやすく共有 |
| 配置内容 | documents / manifests / receipts | documents 内の実体ファイルだけ |
| 配置方法 | 実行ごとの保管フォルダ | 指定先へ実名で配置 |

同名ファイルを再送した場合は同じ場所を更新します。送り先側に版管理があれば、Box / SharePoint / Drive の版、
または S3 / Blob のバージョニングとして履歴を残せます。

## 対応する送り先

| 種類 | 用途 |
|---|---|
| ローカル／ネットワーク共有 | 社内ファイルサーバー、NAS、Box / Drive / SharePoint の同期フォルダ |
| Amazon S3 | バケット内へ実名配置 |
| Google Cloud Storage | バケット内へ実名配置 |
| Azure Blob Storage | コンテナー内へ実名配置 |

## アプリでの使い方

1. 「プラグイン」画面で `upload` を導入し、状態を確認する
2. 「接続先テンプレート」でファイル共有向けの送り先を作る
3. 配置先を指定する
   - ローカル／ネットワーク共有: マウント済みフォルダや同期フォルダ
   - クラウド: S3 / GCS / Azure Blob の保存先
4. 必要なら「配置先サブパス」を指定する
5. フローの送り先に追加し、まず手動実行で配置結果を確認する

配置先サブパスでは `{flow}`、`{date}`、`{datetime}`、`{batchId}` を使えます。未指定なら、指定した保存先の
直下へ実名で配置します。

## 主な設定

| 設定 | 説明 |
|---|---|
| 配置先 | S3 / GCS / Azure Blob のURL、またはローカル／ネットワーク共有のパス |
| 配置先サブパス | 保存先の下に作る任意の階層。フロー名や日付を差し込み可能 |
| 配置対象ファイル名 | 特定ファイルだけを再送するときの絞り込み |
| お試し実行 | 書き込まず、配置先と対象ファイルだけを確認 |

クラウド認証の考え方は [i-repo-archive](i-repo-archive.md) と共通です。スケジュール実行では、対話ログインではなく
サービスアカウントやアクセスキーなど、無人実行に適した資格情報を使ってください。

## 安全設計

- `documents/` に存在するファイルだけを配置します。
- 配置後に読み戻して確認し、`verified:true` のときだけ成功として扱います。
- パスワードやクラウドの秘密値は、コマンド引数ではなく環境変数で渡します。
- ローカルの保管フォルダを消す設定は破壊的操作です。通常はオフのまま使い、必要性を確認してから有効にしてください。

## トラブルシューティング

- **PDF / Excel がない** → フローの抽出形式が「テキストのみ」や「一覧だけ」になっていないか確認する。
- **同期フォルダへ置けない** → 同期アプリが起動済みで、保存先フォルダに書き込み権限があるか確認する。
- **クラウドだけ失敗する** → 「プラグイン」画面で外部CLIと認証の状態を確認する。
- **同名ファイルの履歴を残したい** → Box / SharePoint / Drive の版管理、またはクラウドストレージのバージョニングを有効にする。
