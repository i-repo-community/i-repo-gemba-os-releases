# i-repo-hello 使い方

> 受け取った設定をそのまま表示する、学習・動作確認用のサンプル（macOS・Windows・Linux 対応）

i-repo-hello は、**プラグインの仕組みを学んだり、設定が正しく渡っているかを確かめたりするための練習用サンプル**です。
受け取った引数と、i-repo が渡す `IREPO_*` という設定値（エンドポイントなど）を、加工せずそのまま画面に表示するだけのプラグインです。
本番の保管・送信はしません。

## CLI での使い方

```bash
# 引数や設定がどう渡るかを確認
i-repo hello --foo bar --flag

# i-repo が渡す設定値（エンドポイント等）を確認
IREPO_ENDPOINT=https://example.com i-repo hello
```

## 何の役に立つか

- **新しいプラグインを書く前の見本**: 引数の受け取り方、設定値（`IREPO_*`）の受け取り方、終了の仕方を、いちばん小さい形で確認できます。
- **アプリからの呼び出し確認**: アプリから起動したときに、接続先や認証の情報がきちんと渡っているかの切り分けに使えます。
- **最小プラグインの例**: 自己説明（`--plugin-schema`）を持たない最小構成のプラグインがどう動くか（アプリ上では詳細表示が限られます）を確かめられます。

## プラグインを自作するときの出発点

このプラグインを `~/.i-repo/plugins/i-repo-<名前>` にコピーして作り替えるのが近道です。
配信用のプラグインにするなら、最低限つぎを用意します。

1. `--plugin-schema` で自己説明（受け取る設定・入力・対応OS）を返す
2. `--plugin-healthcheck` で依存や認証がそろっているかを返す（`ok` / `checks[]`）
3. `--plugin-verify` で約束どおりに動くかを返す
4. 本処理は **書き込み → 検証 → 控え（receipt）** の3段で行い、検証結果（`verified:true/false`）を控えに入れる
5. 標準入力から NDJSON を受け取るなら `input: ["stdin-ndjson"]` と宣言する

詳しくは [mongo](i-repo-mongo.md) / [sqlite](i-repo-sqlite.md) の実装が参考になります
（追加の外部ライブラリなし・標準機能だけで書かれています）。

## 技術メモ（仕組み）

> ここから先は仕組みを知りたい人向けです。ふだんの利用では読まなくて問題ありません。

- **引数の受け渡し**: コマンドに付けた引数（`--foo bar` など）は、そのままプラグインへ渡されます。hello はそれを解釈せずに表示するので、何がどう渡ったかをそのまま確認できます。
- **`IREPO_*` 環境変数**: i-repo は接続先などの設定を `IREPO_ENDPOINT` のような環境変数でプラグインに渡します。hello はこれらを一覧表示します。
- **`PATH` の引き継ぎ**: アプリからプラグインを起動するときは、コマンドの探索先（`PATH`）などの実行環境も引き継がれます。アプリ経由でうまく動かないときは、ここが引き継がれているかの確認に hello を使えます。
