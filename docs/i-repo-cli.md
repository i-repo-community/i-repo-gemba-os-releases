# i-Repo CLI とは

> すべての土台になっているコマンドラインツール

i-Repo GEMBA OS（アプリ）が内部で使っているのが **i-Repo CLI** です。
ConMas i-Reporter のデータを**コマンドラインから取り出す・書き出す・自動化する**ための
ツールで、アプリの「送り先プラグイン（SQLite / S3 / MongoDB / BigQuery …）」も、
すべてこの i-Repo CLI のプラグインとして動いています。

- **アプリ（GEMBA OS）だけを使う場合**は、CLI を直接さわる必要はありません。アプリが
  内部で i-Repo CLI を呼び出します。
- **CLI を単体で使う**と、cron / CI / シェルスクリプトなどの既存の自動化に、帳票データの
  取得・エクスポートをそのまま組み込めます。

ソース・詳細: <https://github.com/i-repo-community/i-repo> ／
[デモムービー](https://www.youtube.com/watch?v=fIP9AMn6niE)

## できること

- バッチ取得・日次エクスポートなどの**定型運用**をコマンド一発で
- cron / CI / パイプラインとの連携
- 取り出したデータを、アプリの送り先プラグインで各データ基盤へ配信

## 入手（CLI を単体で使う場合）

```bash
# npm（どの環境でも利用可能）
npm install -g i-repo

# pnpm（セキュリティ重視の環境で推奨）
pnpm add -g i-repo
```

> セキュリティを重視する環境では **pnpm** を推奨します。pnpm は依存パッケージのインストール時
> スクリプトを既定でブロックするため、サプライチェーン攻撃を防ぎやすくなります（npm でも
> `npm install -g i-repo --ignore-scripts` で同等のブロックが可能）。依存は全てバージョン固定済みで、
> npm / pnpm のどちらで入れても取得される依存は同一です。

## 使い始め

```bash
# 対話形式でエンドポイント・ID・パスワードを入力して起動
i-repo
```

接続できたら、帳票の一覧取得やエクスポートをコマンドで実行できます。アプリと組み合わせる場合は、
アプリの画面から「いつ・どの帳票を・どこへ」を設定するだけで、この CLI が裏側で動きます。

## アプリとの関係（まとめ）

| | i-Repo CLI | i-Repo GEMBA OS（アプリ） |
|---|---|---|
| 形 | コマンドライン | 画面（GUI）アプリ |
| 役割 | 土台。データ取得・エクスポート・自動化 | CLI を画面から使い、送り先への配信・確認・AI 連携をまとめる |
| 向き | 自動化・スクリプト連携 | 設定もコードも要らずに使いたい現場 |

送り先プラグインの個別の使い方は[プラグイン使い方集](プラグイン使い方集/)を参照してください。
