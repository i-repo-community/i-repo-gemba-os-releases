# API

この機に貯まった現場蓄積データを、社内LANに常時つながる**読み取り専用のAPI**として公開する画面です（システム節）。
以前「現場蓄積データ」画面にあった「**配る**」機能（共有エクスポート・チケット発行）はここへ移りました。メンバーへの共有は、まずこの「API」を使うのが既定の経路です（同じ社内ネットワークにいる相手なら、トークンを1本渡すだけで最新のデータをそのまま読めます）。

## 稼働のオン・オフとベースURL

- 上部のスイッチで API の稼働を切り替えます。状態は **稼働中 / 停止中 / 待受エラー** の3段階で表示され、待受エラー（ポート衝突など）のときは黙って止まらず警告が出ます。
- 現在のベースURL（`http://localhost:<ポート>/api/v1`）が常に表示されます。既定は**自機（127.0.0.1）のみ**の待受です。
- 「詳細設定」を開くと、**待受ポート**の変更と**LAN に公開する（0.0.0.0）**の切替ができます。LAN 公開は平文 HTTP のため、社内 LAN 内に限定し社外へは持ち出さないでください（Windows では初回にファイアウォールの確認が出ることがあります）。

## トークンの発行・編集・削除

- 「＋ API トークンを発行」から、**宛先ラベル（誰に渡すか）**・**読める範囲（データセット）**・**有効期限（30日／90日／1年）**を指定して発行します。
- 発行すると、ベースURL・トークン・「まず `openapi.json` を読む」案内・Swagger UI の場所をまとめた**接続ガイド**がクリップボードへコピーされます。トークンの全文が画面に見えるのはこの発行直後だけです（以降はマスク表示）。Slack など安全な経路で読み手に渡してください。
- 発行済みトークンの台帳では、**編集**（宛先・範囲・期限を書き換えると、再配布なしで即座に権限へ反映されます）・**失効**・**削除**（失効済み／期限切れの行のみ）ができます。

## curl で試す

コマンドラインからすぐ確認できる3つのコピー可能なコマンド例があります。

- 疎通確認（`GET /health`・トークン不要）
- データセット一覧の取得（`GET /datasets`）
- 件数だけの問い合わせ（`POST /datasets/{id}/query` に `countOnly`）

## Swagger UI（ブラウザで試す）

「Swagger ドキュメントを開く」ボタンで、ベースURLの `/docs` を既定のブラウザで開き、API を探索・Try-it-out できます（API 稼働中のみ配信されます）。

## 従来の共有（非推奨・既定では非表示）

設定の「使い方」欄にある「**従来の共有（snapshot 配布）を表示**」を ON にしたときだけ、この画面の下に**従来の共有**（凍結・非推奨バッジ表示）が現れます。

- **共有エクスポート**: 選んだ配信先（S3 / GCS / Azure / フォルダ）へ、現テナントの蓄積データを共有用に書き出します（自動共有トグルで配信のたび自動更新も可）。
- **メンバーへのチケット発行**: 宛先・読める範囲・有効期限を決めて共有チケット（JWT）を発行し、読み手は[共有ビュー](screen-share.html)にそのチケットを貼るだけで読めます。

同じ社内ネットワークの相手には API を、**拠点外・オフラインの相手**には従来の共有を、という使い分けです。UI は残っているため既存の運用が壊れることはありません。
