# 共有ビュー

アドミンから発行された**従来の共有チケット**（`gemba-share://v1/...`）を、読み手（メンバー）として読み込むための独立した画面です。自分のテナントの「[現場蓄積データ](screen-gemba.html)」画面（自テナントの配信済みデータ）とは**別の読み取り経路**で、共有ソースは書き手側の接続先に紐づきます。

> **v0.30.0 で従来の共有は既定で非表示になりました。** この画面を使うには、設定の「使い方」欄にある「**従来の共有（snapshot 配布）を表示**」を ON にする必要があります（書き手・読み手の双方でこの設定が要ります）。同じ社内ネットワークにいる相手であれば、[API](screen-readapi.html) 画面のトークンを、現場蓄積データ画面の「データソース」欄に**API ソースとして登録**するほうが簡単です。共有ビューは、拠点外・オフラインの相手などネットワークが直接つながらない相手に**チケットを貼って渡す**運用のために残っています。

> スクリーンショット: 準備中（後日追加）

## チケットを貼って追加

- 貼り付け欄にチケット文字列を貼り、「内容を確認」でデコードします（**署名検証はせず表示のみ**）。接続先(host)・共有範囲・宛先・有効期限のプレビューを確認してから保存します。
- 「**取得して現場AIで問う**」を押すと、読む準備（同梱プラグイン確認）→ 保存 → 取得 → 現場AIチャット起動までを1回の操作でまとめて行います。途中で失敗しても保存済みソースは一覧に残り、「読む準備（同梱プラグイン導入）」ボタンで導入してから再開できます。

## 共有ソース一覧

- 登録済みの共有ソースを一覧表示します（名前・接続先(host)・共有範囲・最終取得）。
- 画面を開くたびに各ソースを自動で最新化します（差分取得のため高速。失敗しても閲覧は継続でき、静かな通知に留まります）。
- 行ごとに「**今すぐ取得**」（手動で最新化）・「**このソースを現場AIで問う**」（取得済みのときのみ）・「**削除**」ができます。

## データセット一覧

一覧からソースを選ぶと、そのソースで取得済みのデータセット（タイトル・ID）が表示されます。未取得の場合は「今すぐ取得」で埋まります。
