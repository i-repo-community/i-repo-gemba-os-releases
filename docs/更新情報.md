# 更新情報（What's New）

アプリの主な改善・修正をまとめます。最新の配布物は
[ダウンロード（最新リリース）](https://github.com/i-repo-community/i-repo-gemba-os-releases/releases/latest)

## 2026年7月

### 帳票定義の「改版」に強くなりました

i-Reporter で帳票定義を改版すると定義IDが新しくなりますが、これまでのバージョンは古いIDのままの場所で帳票を見失うことがありました。改版をまたいでも一貫して見つかるようにしました。

- **改版した定義でも帳票一覧が出る（修正）**
  Ontology Studio で定義を開いたとき、改版**前**に作られた帳票が「この定義の帳票一覧（0件）」になっていました。
  改版の前後（すべてのリビジョン）の帳票をまとめて表示します。
- **昔の定義IDでも検索でヒット（修正）**
  帳票定義一覧の検索が、現在のIDと名称しか見ていませんでした。**元ID**でも探せるようにしたので、
  i-Reporter の管理画面で見たIDをそのまま貼り付けて見つかります。
- **配信フローの「定義ID指定」が改版に追従（修正）**
  定義IDで絞った配信フローは、改版後に作られた新しい帳票を拾えなくなることがありました。
  改版の前後すべての帳票を対象にするので、**定義を改版しても配信が漏れません**（同梱プラグインの更新で反映されます。
  「プラグイン」画面から更新してください）。

### 現場スキル ― ベテランの“見方”が、持ち運べるスキルになる（v0.20.0）

v0.19.0 の「業務の重なり」を **現場スキル** として全面刷新しました。書き留めた見方は、アプリの中でも外でも“スキル”として働きます。

- **現場スキル（改名＋進化）**
  複数の帳票を「一緒に見る」ベテランの見方を、AIインタビューで引き出して人が確定すると、**現場AIのスキル**になります。
  現場AIチャットは **「どの現場スキルを使いますか？」** から始まり、スキルを選んで・または自然な質問だけで、
  帳票をまたいだ答えが返ります（必要なときだけスキルが読み込まれる仕組みで、会話は軽いまま）。
- **スキルを書き出す（新機能）**
  確定したスキルはワンクリックで書き出せます（Agent Skills 標準・SKILL.md）。**お使いの Claude Code や Codex で
  そのまま使えます**——このアプリを開いていなくても、現場の見方があなたのAIに宿ります。誰が・いつ確認したかも一緒に持ち出されます。
- 主権は不変です。スキルに入るのは**ラベル・人が確認した意味と見方・照会の手順だけ**。帳票の値や個人情報は含まれません。

### 業務の重なり ― 複数の帳票を“一緒に見る見方”を書き留める（v0.19.0）

複数の帳票定義をまたいで見るベテランの視点を、AIと一緒に書き留めて現場AIに持たせられるようにしました。

- **業務の重なり（新機能）**
  点検記録・作業実績・異常報告など、**別々の帳票を「一緒に見る」まとまり**を作れます（2〜5枚）。
  「なぜ一緒に見るか」と「この重なりの見方（INSTRUCTION）」を書き留め、**人が確認（確定）**すると、
  現場AIが**帳票をまたいだ質問**に答えられるようになります（誰が・いつ確認したかも残ります）。
- **AIインタビューで“見方”を引き出す（新機能）**
  「この帳票たち、現場ではどんな順番・きっかけで一緒に見ますか？」とAIが聞き役になり、
  やりとりから見方の下書きを作ります。ゼロから書かなくても始められます。
- **どこからでも「重なりで試す／聞く」（新機能）**
  Ontology の「現場AIで試す」に加え、**現場AIチャットからも確定済みの重なりで直接聞ける**ようにしました。
  Ontology を開かなくても、ベテランの見方で跨ぎ質問できます。
- **帳票／定義／現場データの一覧をマネージャ準拠で共通化（改善）**
  帳票ライブビュー・Ontology・現場蓄積データの一覧列（ID・帳票名・元ID・REV・状態・登録/更新日時）を
  i-Reporter 入力帳票マネージャに合わせて揃え、状態を分かりやすいラベル表示にしました。
- 主権は不変です。AIに渡すのは**識別子と人が確認した意味・見方だけ**で、帳票の値や個人情報は送りません。

### Ontology の現場AI連携を磨き込み（v0.18.1）

- **意味づけを“この帳票だけ”に絞って渡す（改善）**
  「現場AIで試す」で帳票を持ち込んだときは、**その帳票の確認済みの意味づけだけ**をAIに渡すようにしました。
  現場全体の辞書を毎回送らないので**軽く・速く**なり、回答もその帳票に集中します。
- **新規チャットに前の帳票の意味づけを持ち込まない（修正）**
  まっさらな新規チャットでは、意味づけ（オントロジー）を**自動では載せない**ようにしました。帳票を持ち込んだ
  ときだけ効きます（必要なら手動で含めることも可能）。
- **既存データでも「オントロジー対応」表示がすぐ効く（修正）**
  これまで再配信するまで空だった、現場蓄積データの**「オントロジー対応のみ」絞り込み・意味づけ状態バッジ**が、
  既存の配信データでもすぐ反映されるようにしました。

### Ontology Studio（新機能）— 現場AIチャットに“脳”を与える（v0.18.0）

- **Ontology Studio（新機能）**
  サイドバー上部に **「Ontology Studio」** を追加しました。帳票の項目（クラスター）に**型と意味（現場のノウハウ）**を
  付け、**人が確認したものだけ**を現場AIチャットの根拠にできます。開くと **AI-READY 進捗ダッシュボード**（意味づけ
  済み定義の割合）と、「定義一覧 / 意味づけ済み / 未着手 / 続きから / データカタログ」の入口タイルが出ます。
- **意味づけの流れ**
  帳票定義を選ぶ → サンプル帳票から**型をローカル推論**（値は外に出しません）→ **AIが意味を下書き**（ラベル・型のみ・
  値なし）→ **項目インタビュー**や**INSTRUCTION**（この帳票の運用ルール）→ **人が確認して正本化**。確認済みの内容だけが
  現場AIに渡ります。
- **「この定義を持って現場AIへ」1クリック動線**
  定義一覧の各行や意味づけ画面から、**その帳票にスコープして現場AIチャット**を開けます。**この帳票向けの質問候補**が
  自動で出て、**配信データが混在していても itemId で絞って**（＝正確な件数・傾向）照会します。渡している内容
  （識別子・絞り条件・確認済みの意味）は**チャット画面で確認でき**、辞書は**含める/外す（トークン節約）**を選べます。
- **データカタログ（現場蓄積データ）にオントロジー対応レンズ**
  「オントロジー対応のみ」（定義に紐づく配信）に絞り込み、**意味づけ状態バッジ**を表示。フロー作成に
  **「帳票定義ID（推奨）」**欄を追加し、配信を定義スコープにすると意味づけと対で活かせます。
- **サイドバー再構成**
  「現場AIチャット / Ontology Studio / 概要」を最上段に、以下を「配信フロー（01–05）」「運用・連携」に整理しました。

### i-repo ログビューア（新機能）・設定画面の起動高速化（v0.17.0）

- **i-repo ログビューア（新機能）**
  サイドバーに **「ログ（i-repo）」** を追加しました。i-repo CLI が書く診断ログ（`~/.i-repo/logs/`・直近7日・
  パスワードはマスク済み）を、アプリから**読み取り専用**で閲覧できます。**種類（すべて／コマンド／通信／完了／
  エラー／失敗）でのワンタッチ絞り込み・テキスト検索・当日ログの自動追従（tail）・フォルダを開く**に対応。
  障害解析や操作の確認で、ターミナルを開かずに済みます（アプリの「実行状況」＝配信ごとの記録とは別物です）。
- **設定画面が速く開くように（改善）**
  「設定」を押してから画面が出るまでの待ちを解消しました。枠をすぐ表示して接続設定は届き次第うめる方式にし、
  裏側の情報取得も並行化したので、**非力な PC でも即座に開き**ます。
- **Windows での接続先解決の安定化（修正）**
  一部の環境で `~/.i-repo`（設定・カタログ・ログの保存先）の解決先がずれることがあった問題を解消し、
  i-repo CLI と必ず同じ場所を参照するようにしました。

## 2026年6月

### 帳票ライブビューの絞り込み強化・データ健康診断・安定性向上（v0.16.0）

- **データ健康診断（新機能・試験）**
  「現場蓄積データ」でデータセットを選び、**「健康診断（試験）」**を押すと、配信済みデータを読み取り専用で
  サンプリングし、**欠損・重複・記入ゆれ・鮮度**を点検して**品質スコア（A〜D）**を表示します。さらに、集計結果
  だけを使って **AI が気づきと改善のヒント**を**現場向けのやさしい言葉**で返します（帳票の中身＝個人情報は AI に
  渡しません。ローカルの Claude が必要）。まずは試験的な機能としての公開です。
- **現場蓄積データの読み込みを安定化（修正）**
  データセットを開いたときに、まれに「取得に失敗（切断）」と表示されることがあった問題（読み込み直後の一過性）を
  自動で再試行して解消し、さらに**入力項目が非常に多い帳票を含むデータ**でも最後まで取りこぼさず表示できるように
  しました。
- **帳票ライブビューを条件で絞り込めるように（改善）**
  「帳票ライブビュー」で、**日付範囲（登録/更新）・編集ステータス・公開ステータス・定義ID・ラベルID・
  システムキー・検索語**を指定して、取得する段階で必要分だけに絞れるようにしました。既定は「更新が直近7日」で、
  毎回すべてを取り込まないようにしています。
- **「ラベルID」の初期値を「すべて（全階層）」に（改善）**
  帳票ライブビューの既定が最上位階層だけに絞られて0件になりやすかった点を、**全階層（-9）**を既定にして解消しました。
- **接続テストに失敗したときの原因ガイド（改善）**
  接続テストが失敗したとき、配信先の種別に応じて**よくある原因と対処（再ログイン手順・資格情報の設定など）**を
  表示するようにしました。
- **画面のエラーでアプリ全体が真っ白になりにくく（改善）**
  ある画面で予期しないエラーが起きても、その画面だけにエラー表示を留め、ほかの画面は引き続き使えるように
  しました（「再試行」「再読み込み」も可能）。

### フロー編集の高速化・抽出の初期値見直し・表示の修正（v0.15.0）

- **フロー編集を開いたときの項目表示が速くなりました（改善）**
  フロー編集画面で、抽出フィルタの**ラベル・選択肢などの項目が表示されるまで数秒待たされる**ことがありました。
  項目の表示に必要な情報を先に読み込み、接続確認などの重い処理は後回しにすることで、**項目がすぐ出る**ように
  しました。プラグインの状態表示（バッジ）はこれまでどおり少し遅れて埋まります。
- **抽出の「ラベルID」の初期値を「すべて（全階層）」に（改善）**
  抽出フィルタの「ラベルID」を未指定のままだと、対象が**最上位階層のフォルダ（ラベル）だけ**に絞られ、
  運用によっては**対象帳票が0件**になり、分かりにくいエラー（権限エラー code=3）で抽出が失敗することが
  ありました。初期値を**「すべて（全階層）」＝ -9** にして、新しく作るフローが既定で全階層を対象にするように
  しました（この変更は「プラグイン更新」からも届きます）。
- **帳票ライブビューで取得中に「該当なし」が一瞬出る問題を修正（修正）**
  帳票ライブビューを開いた直後、まだ読み込み中なのに**「条件に一致する帳票がありません」が一瞬表示される**
  ことがありました。取得中は**スピナー＋「取得中…」**を出すようにし、誤解を防ぎます（既存の一覧を「↻ 更新」
  する場合は、取得が終わるまで前の一覧を保持します）。あわせて出所説明の表記を、全件とは限らない実態に合わせて
  「現在取得した帳票一覧」に見直しました。

### 更新のお知らせを分かりやすく（v0.14.0）

- **新しいバージョンが出たら起動時にお知らせ（改善）**
  アプリの新版が公開されると、起動時に**ポップアップでお知らせ**するようにしました。「更新する」（ダウンロード
  ページを開く）か「後で」を選べます。「後で」を選んだ版は**もう通知しません**（さらに新しい版が出れば、また
  一度だけお知らせ）。サイドバーの「↑ 新版あり」表示は従来どおり残ります。
- **プラグインの更新も起動時にお知らせ＆ワンクリック更新（改善）**
  アプリに同梱された新しいプラグインがあると、起動時にお知らせし、**その場で一括更新**できます。サイドバーにも
  「↑ プラグイン更新あり」を表示します。これまでは「プラグイン」タブを開かないと気づけず、**古い版のまま動いて
  「直っていない」と誤解する**ことがあった点の対策です（公開ミラーにさらに新しい版があるときは、そちらの更新に譲ります）。

### クラウドへの自動配信の認証を安定化（v0.13.0）

- **スケジュール／Webhook での無人配信が認証切れで止まる問題に対応（改善）**
  スケジュールや着信 Webhook で GCS / S3 へ自動配信するとき、対話ログイン（`gcloud auth login` など）は
  時間で失効し、無人実行では再ログインできず**無音で失敗**することがありました。失効しない
  **サービス資格情報**（GCS＝サービスアカウント鍵、S3＝IAM アクセスキー、Azure＝接続文字列／SAS）を
  接続先に設定して、無人でも安定して配信できるようにしました。資格情報の入力欄は接続先テンプレートに
  自動で表示され、秘密の値は**伏字で保存**・実行時は**環境変数で安全に**渡されます（コマンドやログに平文で
  出ません）。設定のしかたは「プラグイン使い方集 → i-repo-archive → 無人運用での認証」を参照してください。

### 使いやすさ・分かりやすさの向上（v0.12.0）

- **フロー作成の配信先がすっきり1か所に（改善）**
  配信先の指定を1つのリストにまとめました。1件なら単一配信、2件以上にすると自動で「同時配信」になります（同じ接続先の重複選択も防止）。設定内容が一目で分かるよう、`i-Reporter → 抽出 → 配信先` の流れを示す簡単な図も表示します。
- **画面名を「帳票ライブビュー」に変更**
  旧「データ確認」を、毎回 i-Reporter から最新を取得して見るという性質に合わせて「帳票ライブビュー」に改名しました。
- **検索用の配信先だけのときは取り込みを自動で軽量化（改善）**
  配信先が検索系（Elasticsearch・データベース等）だけで成果物（PDF）を使わない場合、抽出時に PDF 取得を自動で省いて、取り込みを速く・軽くします。
- **表示の言葉を分かりやすく／多言語表示を整備（改善）**
  開発者向けの専門用語を画面から減らし、用語を統一しました（例：「receipt」→「配信レポート」、「watermark」→「前回の更新時刻」）。設定画面なども英語表示にきちんと切り替わるようにしました。

### 配信基盤と AI 連携の強化（v0.11.0）

- **1 つのフローから複数の配信先へ同時に送れるように（複数配信先・新機能）**
  1 回の抽出で、検索向けストア（Elasticsearch など）と保管向けストア（S3 / GCS / Azure / ローカル）へ
  まとめて配信できる「複数配信先」をフロー設定に追加しました。実行状況では配信先ごとの成否を確認できます。
  全配信先が成功したときだけ次回開始位置（取り込み済みの目印）を進めるため、一部失敗時の取りこぼしが起きません。
- **複数配信先に送ったデータを現場 AI / カタログから見つけて読み返せるように**
  複数配信先で送った先（検索ストアなど）が、これまで実行台帳には記録される一方でデータ索引（カタログ）には
  載らず、現場 AI からの発見対象になっていませんでした。配信成功後に各配信先を索引へ登録し、単一配信先と
  同じように発見・読み返しできるようにしました。クラウド保管のフォルダ名にフロー名などの差し込みを使う場合も
  正しい接続先として索引化します。
- **同梱プラグイン／CLI を更新（i-repo 2.14.0・archive 0.6.2）**
  クラウド保管（アーカイブ）プラグインを更新しました。配布アプリの「同梱プラグイン導入」やプラグイン更新は
  この版を配ります。

### 安定性・使い勝手の向上（v0.10.0）

- **クラウドの認証切れを分かりやすく案内**
  S3 / GCS / Azure（aws / gcloud / az）の認証が切れていると、これまでは原因の分かりにくいエラーで止まって
  いました。認証切れを検知して、「どのコマンドで再ログインすればよいか」を添えた平易なメッセージを表示する
  ようにしました。
- **クラウドから取り戻したデータを現場 AI チャットでも読めるように（修正）**
  クラウド（S3 / GCS / Azure）から取り戻したアーカイブを、現場 AI チャットが「ローカルに無い（自動掃除済み）」
  と誤判定して中身を読めないことがありました。取り戻し時に必要な索引を復元し、確実に読めるようにしました。
- **OS スケジュール登録の取りこぼしを防止（修正）**
  着信 Webhook が無効のまま、または対象フローが未許可のまま OS スケジュール（定期実行）に登録できてしまい、
  定刻になっても**何も起きない（無音で失敗する）**ことがありました。登録時に前提（着信 Webhook の有効化・
  フローの起動許可）を確認し、足りなければ理由を示して登録を止めるようにしました。あわせて macOS でも定期
  実行の前回結果（失敗コード）をスケジュール画面に表示し、失敗に気づけるようにしました。
- **画面まわりの整理**
  画面アイコンの見た目を統一し、画面上部の説明バナーを常時表示から折りたためる控えめな表示に変えました。

### 信頼性・安全性の向上（v0.9.0）

- **接続先（テナント）の取り違え防止を強化**
  配信のたびに「実際にどの i-Reporter 接続先から取得したか」を、i-repo CLI 自身の申告（来歴）に基づいて
  記録・確認するようにしました。別の接続先のデータを取り違える事故を、仕組みとして防ぎます。接続先が
  はっきりしないまま実行されるフローは、安全のため自動で停止します（接続先を確定してから再実行してください）。
- **接続先の切替まわりを整理**
  以前の「接続先が切り替わりました」通知は、上記のより確実な仕組みに置き換えて廃止しました。
- **内部基盤の一本化**
  プラグイン・データアクセス仕様を i-repo CLI 側に一本化（同梱）し、二重管理をなくしました（通常の操作・
  動作に影響はありません）。

### 新機能

- **現場 AI チャットが画面を移動しても消えない**
  現場 AI チャットの会話が、別の画面（実行状況・現場蓄積データ など）へ移って戻っても**そのまま続く**ように
  しました。回答の生成中に他の画面を確認しに行っても、戻れば続きが見られます（接続先を切り替えたときだけ
  リセットします）。
- **データや実行状況から、その場で AI に相談**
  「現場蓄積データ」のデータセット／個別の帳票、「実行状況」の実行詳細に**「🤖 AIに相談」ボタン**を追加。
  押すと、その対象（どのデータセット・帳票・実行か、絞り込み条件）を引き継いだ状態で現場 AI チャットへ移動し、
  すぐ質問できます。引き継ぐのは識別子・条件だけで、**帳票の中身や個人情報は送りません**（AI が必要に応じて
  読み取り専用で取得します）。
- **アーカイブの保存先フォルダ名を自由に**
  S3 / GCS / Azure / ファイルサーバーへアーカイブを送るとき、保存先の**フォルダ名を自由に決められる**ように
  しました（これまでは実行ごとの英数字IDフォルダ固定）。配信先設定の「保存先サブパス」に入力でき、
  `{date}`（日付）・`{flow}`（フロー名）・`{datetime}`（日時）・`{batchId}` の**差し込みチップ**をクリックで
  挿入できます（例「点検記録/{flow}/{date}」→ `点検記録/設備点検/2026-06-23/…`）。
- **クラウドに送ったデータを取り戻す**
  S3 / GCS / Azure へ配信したアーカイブを、実行状況の画面から**「☁ クラウドから取得」ボタン**でローカルへ
  取り戻し、中身（PDF 等）をプレビュー／保存できるようにしました（読み取り専用＝送信先は変更しません）。
  ローカルを掃除した後でも、いつでも取り戻して確認できます。
- **接続先（テナント）ごとに表示を分離**
  複数の i-Reporter 接続先を切り替えて使う運用で、**選んだ接続先の情報だけ**が全画面（概要・フロー作成・
  実行状況・スケジュール・データ確認・現場蓄積データ・現場 AI チャット）に表示されるようにしました。
  接続先を切り替えると、画面・チャットがその接続先の内容に切り替わります。**別の接続先のデータが混ざって
  見えることはありません**（接続先が確認できない記録は、明示的に現在の接続先へ紐付けるまで表示しません）。
- **新しいバージョンのお知らせ**
  アプリに**新しい版が公開されると、サイドバーに「↑ 新版 vX.Y.Z あり」**と表示されます。
  クリックするとダウンロードページが開きます（自動更新はしません・お知らせのみ）。これで
  最新版を見逃しにくくなります。
- **定期実行（スケジュール）**
  フローを時刻・曜日で**自動実行**できるようになりました。フローで頻度（毎日／毎週／間隔）を
  決め、スケジュール画面のカレンダーから OS のスケジューラ（Windows: タスクスケジューラ /
  macOS: launchd / Linux: cron）へ登録します。
- **クラウド転送に Google Cloud Storage / Azure Blob を追加**
  これまでの Amazon S3 に加え、アーカイブを **GCS / Azure Blob Storage** へもアップロードできます。
- **アーカイブをローカル/ファイルサーバーへも保存**
  クラウド（S3 / GCS / Azure）だけでなく、**指定フォルダやファイルサーバー（ネットワーク共有）**へ
  アーカイブを保存できるようになりました。接続先テンプレートで「archive」を選び、保存先のパス
  （例: 共有フォルダ）を入れるだけ。専用の外部ツールは不要です。
- **現場 AI チャット（Claude / Codex 対応）**
  アプリの中で、現場データについて **AI に日本語で質問**できます（読み取り専用。お手元の AI を
  そのまま使う形）。**Claude（Claude Code）** に加えて **Codex（OpenAI Codex）** も選べるように
  なりました（画面上部で切り替え）。Codex は安全のため読み取り専用で動きますが、ツールの絞り込みが
  Claude より緩いため **試験的対応** の位置づけです（常用は Claude をおすすめ）。
- **テキストのみ取得**
  PDF / Excel のファイルは取らず、**項目の入力値（テキスト）だけ**を軽く取り込むモードを追加。
  検索・分析が目的のときに高速・軽量です。

### 改善

- **「現場AIチャットへ」の導線を整理**
  データ／実行から現場 AI チャットへ移動するボタンの表記を **「現場AIチャットへ」** に統一しました。
  また、実行が**進行中（実行中）のときはこのボタンを出さない**ようにしました（結果が出てから相談できます）。
- **配信先を種別ごとに整理**
  送り先を「データベース / データレイク / クラウドDWH / 検索 / アーカイブ」などの種別で
  グループ表示し、目的の送り先を見つけやすくしました。
- **「プラグイン」タブの使い勝手**
  表示を高速化し、各プラグインの「フォルダを開く」「アンインストール」を追加しました。
- **プラグインの更新に気づける**
  アプリに同梱された新しい版がインストール済みより新しいとき、「プラグイン」タブで
  「更新あり」を表示するようにしました（ネット接続なしでも判定）。ワンクリックで最新へ更新できます。
  ※ 例えばアーカイブの送り先に Google Cloud Storage / Azure Blob / ファイルサーバーが
  増えても、更新すれば選べるようになります。
- **状態バッジから対処へ**
  「版が変わった」「出所が確認できない」などのバッジを押すと、対処方法やマニュアルへ案内します。
- **使い方ガイドを刷新**
  専門用語を整理し、よりわかりやすい説明に。仕組みの詳細は各ページ末尾の「技術メモ」にまとめています。
- **接続先の切り替え・復活を安全に・気づける形で**
  i-Reporter の接続先が **切り替わった**ときに気づけるよう、画面でお知らせ＋ワンクリックで各フローを
  新しい接続先へ付け替えできるようにしました。接続先に **つながらない**間は、データ確認画面の
  削除・紐付けなどの操作を自動で保留し（うっかり別サーバーの記録を消さないように）、**復活すると
  自動で元どおり**になります。接続先は名前で表示し、別サーバー由来・出所不明の記録を区別します。
- **「いまの接続先」ごとの表示に統一**
  概要・フロー作成・実行状況・スケジュール・着信 Webhook を、**いまつないでいる接続先のもの**に
  絞って表示するようにしました。接続先を切り替えると、これらの画面がいっせいにその接続先の内容へ
  切り替わります。概要の上部には**接続先のドメイン**を表示し、どの接続先の状況かが一目で分かります。
  （接続先テンプレート・クリーンアップ設定・プラグインなどは接続先に依存しないため全体表示のままです。
  すべての実行履歴は「実行状況」画面で確認できます。表示の出し分けのみで、データは消えません。）
- **詳細ドロワーを広く**
  データ確認などの詳細パネル（右からせり出すドロワー）の横幅を大きくし、情報を見やすくしました。

### 修正

- **配信済みなのに「データが無い」と表示される問題を修正（重要）**
  i-Reporter の接続先によっては、配信が成功しているのに「現場蓄積データ」や「現場 AI チャット」で
  データセットが 1 件も出ない（AI が「データがありません」と答える）ことがある問題を修正しました。
  接続先の識別方法のずれが原因で、本番のサーバー設定でほぼ常に起きていました。**過去に配信済みのデータも、
  再配信なしでそのまま見えるようになります。**
- **BigQuery への配信を安定化**
  すでに用意済みのデータセットに送るとき、配信が失敗してしまうことがある問題を修正しました。
- **接続先の切り替え時にログインが保存できない問題を修正**
  接続先を切り替えるとき、「接続テストは成功するのに『検証して保存』で認証に失敗する」現象を
  解消しました。テストと保存が同じ確認を行うようになり、入力した接続先・ユーザー・パスワードを
  そのまま検証して保存します（誤った情報を入れても、動作中の接続設定は壊しません）。
- **Windows で現場 AI チャット（Codex）がデータを読めない問題を修正**
  Windows で Codex を選んだとき、AI が現場データの読み取りツールを使えない／呼び出しが取り消される
  問題を解消しました（読み取り専用は維持）。
- **Mac でアプリから i-repo CLI が見つからない問題を修正**
  i-repo CLI を入れているのに「CLI が見つかりません」と表示されることがある問題を解消しました。
  アプリのアイコンから起動した場合でも、nvm / pnpm / Homebrew などで入れた CLI を正しく見つけます。
- **接続先が未設定のときのダッシュボード表示を整理**
  接続先（i-Reporter）が未設定のときは、概要画面で連携状況の数字を出さずに「接続先を設定する」案内を
  表示するようにしました（これまでの実行履歴は消えません・表示の出し分けのみ）。
- **Mac でアプリから Node.js が「未検出」になる問題を修正**
  Node.js（22.5 以上）を入れているのに、アプリのアイコンから起動すると現場 AI チャット／エージェント
  接続の準備状況で「未検出」と表示されることがある問題を解消しました（i-repo CLI の検出と同じ仕組みに
  そろえ、nvm / pnpm / Homebrew などで入れた Node.js も見つけます）。
- **長い帳票名のアーカイブ取得が失敗する問題を修正**
  帳票名が長い（日本語などを含む）と、PDF/Excel の取得が「name too long」エラーで失敗することがある
  問題を解消しました。あわせて、アーカイブ内のファイル名を **「帳票ID_帳票名」で統一**し、一覧で
  見つけやすくしました。
- **破壊的操作の確認で同じ項目が重複表示される問題を修正**
  アーカイブのクリーンアップ確認ダイアログで、同じ注意（クリーンアップ）が複数回表示される問題を
  解消しました。

### 配布

- **macOS アプリを署名・公証（Gatekeeper 警告が出ない）**
  macOS 版を **CIMTOPS の Developer ID で署名し、Apple の公証（notarization）**を通すようにしました。
  初回起動時の「**悪意のあるソフトウェアか確認できません**」という Gatekeeper 警告が**出ずにそのまま開けます**。
- **Windows はオフライン同梱版を既定に**
  ネットが弱い／無い現場でも確実に起動できます（軽量版・要ネットも選べます）。
- 公開のリリースページから最新版を入手できます。Windows のコード署名は申請手続き中のため、完了までは
  初回起動時に SmartScreen 警告が出ることがありますが、[安全な開き方](manual-setup.html#install-warning)で起動できます。
