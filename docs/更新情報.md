# 更新情報（What's New）

アプリの主な改善・修正をまとめます。最新の配布物は
[ダウンロード（最新リリース）](https://github.com/i-repo-community/i-repo-gemba-os-releases/releases/latest)
から入手できます。

## 2026年7月

### この機の蓄積データを社内LANの「API」として公開できるようになりました（v0.30.0）

これまでチケットを発行して渡していた現場蓄積データの共有を、**社内LANに常時つながる読み取り専用のAPI**
としても使えるようにしました。システムの節に新しく「API」ページが加わり、トークンの発行・編集・削除、
つなぎ方の案内（ベースURL・トークン・curl の例）、ブラウザで試せる Swagger UI（`/docs`）をここで扱えます。
現場の外からも、社内の他システムからも、決まったトークンだけで安全にこの機のデータへ触れられます。

### 「読み手」「読み書き」を使い方として切り替えられるようになりました（v0.30.0）

設定の「使い方」で、**ためる側（書き手）**と**読む側（読み手）**を選べるようにしました。読む側を選ぶと、
接続先・フロー・スケジュールなどのシステム機能が隠れ、現場AIチャットと蓄積データの画面だけが残ります。
自分の現場のデータを送り出す側か、他の現場のデータを読みに行く側かで、見える画面を必要な分だけに絞れます。

### 他の機・他システムのAPIを「ソース」として登録し、横断で検索できるようになりました（v0.30.0）

書き手から受け取った接続ガイド（ベースURLとトークン）を貼り付けるだけで、その API を「ソース」として
登録できるようになりました。現場蓄積データの画面でソースを切り替えて見るほか、**すべてのソース**を選ぶと
複数の現場・システムをまたいで横断検索できます。検索結果からは成果物のダウンロードもでき、現場AIチャットも
登録したソースに向けて質問できます。

### 従来の共有（チケット発行・貼って取得）は既定で隠れるようにしました（v0.30.0）

v0.29.1 で「共有が2動作になった」とお伝えしましたが、今回からは新しい「API」が共有の正面になり、
従来のチケット共有・共有ビューは既定では表示されなくなりました（設定の「共有」欄でオンにすればこれまで
通り使えます。UI は残っており、既存の運用が壊れることはありません）。

### 同梱の i-repo CLI を 2.34.0 に更新しました（v0.30.0）

テナントを指定した状態での読み取り（tenant-scoped native）に追随しました。

### fan-out フローで Elasticsearch 等の検索ができない不具合を修正しました（v0.30.0）

配信先に検索エンジン（Elasticsearch など）だけを接続してあるフローで、必要な接続情報が見つからず検索が
失敗する不具合を修正しました。

### 共有チケットでの GCS 取得に失敗する不具合を修正しました（v0.30.0）

チケット経由で GCS から取得する際、鍵ファイルの渡し方が誤っており失敗するケースがありました。修正済みです。

### 共有が「2動作」になりました（v0.29.1）

チームへの現場蓄積データの共有を、**管理者はチケットを発行して渡すだけ／メンバーは貼って1ボタン**の
2動作に整理しました。

1. **管理者はチケットを発行して渡すだけ**：現場蓄積データの共有欄に「＋チケットを発行」を追加しました。
   宛先・読める範囲・有効期限（30日／90日／1年）を決めると、チケットが切符の形で表示されコピーできます
   （全文が見えるのはその場限りで、台帳には残りません）。Slack などでそのまま相手に渡してください。
   GCS でチケットを発行するには、読み手用のサービスアカウント（メール）を用意し、発行に使う接続の資格に
   その SA 限定の鍵管理権限（roles/iam.serviceAccountKeyAdmin）を付与しておきます。発行フォームにメールを
   入力（記憶されます）。発行に失敗した場合は、エラーにプラグインの実メッセージ（実因）を表示します。
2. **発行台帳**：誰に・どの範囲を・いつまで渡したかを一覧でき、1枚ずつ**失効**できます（保存先の種類に
   より失効できないものもあります）。
3. **自動共有**：配信が成功するたびに共有データを自動更新します（トグル1つで切り替え、最終共有の時刻と
   世代を確認できます）。
4. **メンバーは貼って1ボタン**：受け取ったチケットを共有ビューに貼ると、確認画面が1回出た後
   「取得して現場AIで問う」で保存・取得・チャット起動まで自動で進みます。以後は共有ビューを開くたびに
   自動で最新化され、チャットには「〈時刻〉更新のデータで答えています」と表示されます。
5. 同梱の i-repo CLI を 2.33.0 に更新しました（チケットの発行・失効への対応、GCS 読み取りの修正を含みます）。

### 現場AIチャットの応答を大幅に軽く・速くしました（v0.28.0）

同じ質問にかかるトークン量が約5分の1になりました（実測：「どんなデータがある？」で 129,612 → 24,881）。
中身は3つです。①データの説明（describe）が既定で要約になり、必要なときだけ全文を取るようにしました
（同梱 i-repo CLI 2.32.0）。②チャットで使わない内部ツールの定義を読み込まないようにしました。③回に
よって応答が重くなるブレ（AI 側の実験機能）を無効化し、安定させました。推論は各自のアカウント課金
なので、コスト・速度の両方に効きます。

### 帳票の実体だけを、Box などのフォルダにそっと置く「配置」ができます（v0.27.0）

送り先が **Box / Dropbox / Google Drive / SharePoint** のように**人がフォルダを開いて見る**場合に向けた、
新しい配信先の性格 **「配置（placement）」**を追加しました。「保管（archive）」が送り先に管理用の裏方ファイル
（目録・受領）や実行ごとの英数字フォルダを作るのに対し、配置は **帳票の実体（PDF / 写真 / Excel）だけを、
指定したフォルダの直下へ実名のまま**置きます。人が見るフォルダを散らかさず、取引先にもそのまま渡せます。

- **使い分けは送り先で**：S3 / GCS / Azure など**機械の物置**は従来どおり「保管」、**人が見るファイル共有**は
  「配置」。フロー編集の図では「保管」「配置（実体だけ）」のバッジで区別できます。
- **その代わりの割り切り**：配置は **AI での読み返し・検索はできず、クラウドからの取り戻しもありません**
  （記録だけを残す用途）。「何を・いつ・どこへ送ったか」はアプリの台帳が覚えています。同名は上書きです。
- **Box などへは同期フォルダ経由**：各社の同期クライアントが作るローカル共有フォルダを宛先に指定します。

> 「自由なフォルダ名にしたい」「検索は要らない」だけなら、これまでの「保管」でも同じことができます。
> 配置を選ぶ意味は「**送り先を汚さず、人が使えるフォルダにする**」点にあります。

### 現場AIチャットが、お使いの Claude の契約形態に自動で合わせます（v0.26.0）

現場AIチャットは、パソコンにインストール済みの `claude` をそのまま使います（アプリは認証を肩代わりしません）。
今回、Claude を **サブスク（ログイン）／Anthropic API キー／Amazon Bedrock／Google Vertex／Microsoft Foundry**
のどれで設定していても、**同じ設定を自動で引き継いで**チャットが動くようになりました。

- **クラウド経由でもそのまま使える（新機能）**: これまでは、ターミナルの `claude` は Bedrock 等で動くのに、
  アプリのチャットだけ設定が引き継がれず動かない・黙ってサブスクに切り替わる、といったことがありました。
  アプリが起動時に**あなたが `claude` 用に済ませた設定を検出して引き継ぐ**ので、そのまま会話できます。
- **どの経由で動いているか一目で分かる（改善）**: クラウドや API 経由のときは、チャット上部に
  **「Amazon Bedrock」などのプロバイダ名**が小さく表示されます（サブスクのときは表示しません）。
- **認証はこれまでどおりお使いの仕組みで**: 認証情報はアプリが預かりません（Bedrock は AWS、Vertex は
  Google、Foundry/API はキー）。準備状況パネルに、経由ごとの整え方のヒントが出ます。

### 基本プラグインを最初の起動で自動導入（v0.26.0）

インストール直後の初回起動で、現場でまず使う **保管（archive）／SQLite／削除** の3つのプラグインを
**自動で導入**するようにしました（以前は手動導入が必要で、まっさらな状態だと保管も削除もできませんでした）。
その他の送り先はこれまでどおりプラグイン画面から追加できます。

### 現場AIチャットが「集計」に強くなりました（v0.25.0）

「先月の指摘を**月別**に」「**削除区分ごと**の内訳」「点検**種類の数**」「**合計・平均**」といった集計の質問に、
現場AIチャットが**全行を取り出さずに配信先側で集計**して答えるようになりました。

- 帳票が多くても**軽く・速く**、消費トークンも大幅に削減されます（従来は全件を取り出して数えるため、件数が
  多いと答えられませんでした）。
- 集計を**勝手に勧めることはなく**、集計を聞かれたときだけこの方法で答えます（**読み取り専用は不変**）。
- 対応送り先は **SQLite / Elasticsearch** から始め、現在は **MongoDB / Parquet / BigQuery** も対応（プラグイン更新で入手できます）。

### 帳票ライブビューから、i-Reporter アプリでそのまま開ける（v0.23.0）

帳票ライブビューの帳票IDの横に **i-Reporter アイコン**が付きました。クリックすると、その帳票を
**i-Reporter アプリ**で開けます（アプリのインストール／ログインが必要）。ID の文字は従来どおり
**Web ビューア（ConMas Manager）**を開くので、用途で使い分けられます。

- **アプリで開く（新機能）**: 帳票IDの横のアイコンから、その帳票を i-Reporter アプリで直接開きます。
  安全のため、開けるのは**接続先サーバ自身が示す正規のアプリ起動リンクだけ**（認証情報は含めません）。
- **現場AIの文脈を軽く・正確に（改善）**: 定義や「業務の重なり」を現場AIに渡すとき、これまで帳票の

- Ontology Studio の説明文も、「現場のノウハウを書き留め、現場スキルへ育てる」流れが伝わる文言に磨きました。

### 帳票の「削除フロー」― 配信済みを狙って安全に消す（v0.22.0）

i-Reporter サーバ側の帳票削除を、配信フローと同じ「フロー」の考え方に一本化しました。「1つの条件＝1つの削除フロー。即時実行も定期実行も同じフローのトリガー違い」です。

- **削除フロー管理（新機能）**
  削除画面が「削除フロー」の一覧＋編集に変わりました。条件（相対／絶対）・スケジュール・直近の実行が一目で分かり、
  その場で ▶即時実行、または日次／週次のスケジュールに載せられます。
- **消せるのは「配信済み」だけ（安全設計）**
  削除対象は常に「いま接続している現場で、外部にコピーがある（配信済みの）帳票」に限定します。未配信を消すと
  外部にもコピーが残らずデータ消失になるため、実行のたびに「配信済み ∩ 条件」を backend が再解決してから消します。
- **「Nヶ月／日より古いものを消す」を直感的に（新機能）**
  更新／登録が一定期間より古い帳票を、相対条件で指定できます。条件は相対のまま保存され、定期実行のたびに
  「その時点」を基準に計算し直すので、基準が古いまま凍りません。
- **無人実行でも安全（安全設計）**
  定刻の自動実行でも、実行のたびに「削除が有効か」「接続先が意図した現場か」を再確認し、違えば中止します。
  1回の削除は最大1000件で頭打ち（超過は次回に持ち越し）。条件ゼロ（全件削除）になる指定は保存前に拒否します。

### 選んだ帳票を、そのまま現場AIに聞ける

蓄積データの意味づけ（スキル）が「見方＝レンズ」なら、そこに **実際のデータ＝被写体** を重ねられるようにしました。
一覧で「これ」と選んだ帳票を、そのまま現場AIチャットの主題として渡せます。

- **現場蓄積データで帳票を選んで聞ける（新機能）**
  行をチェックして「選んだ帳票について現場AIに聞く」。選んだ実レコードが会話の主題になり、AIがその中身を
  読み取って答えます。値・個人情報は渡さず識別子だけを渡す仕組み（AIが読み取りツールで自分で参照）は不変です。
  選べるのは少数精鋭（キュレーション）に意味があるので上限20件。
- **配信実績クリックで、その帳票のデータへ一直線（新機能）**
  帳票ライブビューの「配信実績」をクリックすると、その配信先の現場蓄積データが開き、**その帳票が選択済み**で
  表示されます。そのまま「聞く」か、周りの帳票も足して聞けます。ライブビュー・蓄積データ・現場AIが一本の導線に。

### スケジュール画面で「何が動いているか」が分かる（改善）

これまで数字だけだったスケジュール画面を、**どの自動処理（配信フロー・削除フロー等）がいつ動くか**が

### 現場（テナント）の混ざりを解消しました（v0.21.0）

同じ配信先（データベースや検索の入れ物）に、時期をまたいで複数の i-Reporter から届いたデータが同居し、読み返しに**別の現場の帳票が混ざって見える**ことがありました。**いま接続している現場の分だけ**を読み返すようにしました。

- **別の現場のデータが混ざらない（新機能）**
  現場AIチャット・現場蓄積データの読み返しが、接続中の現場（エンドポイント）の分だけに自動で絞られます。
  配信のたびに「どの現場から来たか」の印が記録され、それで厳密に切り分けます。過去に配信した分は、
  その現場で**もう一度配信する**と印が付いて表示に戻ります（同梱プラグインの更新が必要です。「プラグイン」画面から更新してください）。
  ※テナント分離が有効な配信先では、上級者向けの「ネイティブ検索」は当面ご利用いただけません（通常の絞り込み検索をお使いください）。

- **画面まわりの整理**
  画面アイコンの見た目を統一し、画面上部の説明バナーを常時表示から折りたためる控えめな表示に変えました。

### 信頼性・安全性の向上（v0.9.0）

- **接続先（テナント）の取り違え防止を強化**
  配信のたびに「実際にどの i-Reporter 接続先から取得したか」を、i-repo CLI 自身の申告（来歴）に基づいて
  記録・確認するようにしました。別の接続先のデータを取り違える事故を、仕組みとして防ぎます。接続先が
  はっきりしないまま実行されるフローは、安全のため自動で停止します（接続先を確定してから再実行してください）。
- **接続先の切替まわりを整理**
  以前の「接続先が切り替わりました」通知は、上記のより確実な仕組みに置き換えて廃止しました。
- **内部基盤の一本化**
  プラグイン・データアクセス仕様を i-repo CLI 側に一本化（同梱）し、二重管理をなくしました（通常の操作・
  動作に影響はありません）。

### 新機能

- **現場 AI チャットが画面を移動しても消えない**
  現場 AI チャットの会話が、別の画面（実行状況・現場蓄積データ など）へ移って戻っても**そのまま続く**ように
  しました。回答の生成中に他の画面を確認しに行っても、戻れば続きが見られます（接続先を切り替えたときだけ
  リセットします）。
- **データや実行状況から、その場で AI に相談**
  「現場蓄積データ」のデータセット／個別の帳票、「実行状況」の実行詳細に**「🤖 AIに相談」ボタン**を追加。
  押すと、その対象（どのデータセット・帳票・実行か、絞り込み条件）を引き継いだ状態で現場 AI チャットへ移動し、
  すぐ質問できます。引き継ぐのは識別子・条件だけで、**帳票の中身や個人情報は送りません**（AI が必要に応じて
  読み取り専用で取得します）。
- **アーカイブの保存先フォルダ名を自由に**
  S3 / GCS / Azure / ファイルサーバーへアーカイブを送るとき、保存先の**フォルダ名を自由に決められる**ように
  しました（これまでは実行ごとの英数字IDフォルダ固定）。配信先設定の「保存先サブパス」に入力でき、
  `{date}`（日付）・`{flow}`（フロー名）・`{datetime}`（日時）・`{batchId}` の**差し込みチップ**をクリックで
  挿入できます（例「点検記録/{flow}/{date}」→ `点検記録/設備点検/2026-06-23/…`）。
- **クラウドに送ったデータを取り戻す**
  S3 / GCS / Azure へ配信したアーカイブを、実行状況の画面から**「☁ クラウドから取得」ボタン**でローカルへ
  取り戻し、中身（PDF 等）をプレビュー／保存できるようにしました（読み取り専用＝送信先は変更しません）。
  ローカルを掃除した後でも、いつでも取り戻して確認できます。
- **接続先（テナント）ごとに表示を分離**
  複数の i-Reporter 接続先を切り替えて使う運用で、**選んだ接続先の情報だけ**が全画面（概要・フロー作成・
  実行状況・スケジュール・データ確認・現場蓄積データ・現場 AI チャット）に表示されるようにしました。
  接続先を切り替えると、画面・チャットがその接続先の内容に切り替わります。**別の接続先のデータが混ざって
  見えることはありません**（接続先が確認できない記録は、明示的に現在の接続先へ紐付けるまで表示しません）。
- **新しいバージョンのお知らせ**
  アプリに**新しい版が公開されると、サイドバーに「↑ 新版 vX.Y.Z あり」**と表示されます。
  クリックするとダウンロードページが開きます（自動更新はしません・お知らせのみ）。これで
  最新版を見逃しにくくなります。
- **定期実行（スケジュール）**
  フローを時刻・曜日で**自動実行**できるようになりました。フローで頻度（毎日／毎週／間隔）を
  決め、スケジュール画面のカレンダーから OS のスケジューラ（Windows: タスクスケジューラ /
  macOS: launchd / Linux: cron）へ登録します。
- **クラウド転送に Google Cloud Storage / Azure Blob を追加**
  これまでの Amazon S3 に加え、アーカイブを **GCS / Azure Blob Storage** へもアップロードできます。
- **アーカイブをローカル/ファイルサーバーへも保存**
  クラウド（S3 / GCS / Azure）だけでなく、**指定フォルダやファイルサーバー（ネットワーク共有）**へ
  アーカイブを保存できるようになりました。接続先テンプレートで「archive」を選び、保存先のパス
  （例: 共有フォルダ）を入れるだけ。専用の外部ツールは不要です。
- **現場 AI チャット（Claude / Codex 対応）**
  アプリの中で、現場データについて **AI に日本語で質問**できます（読み取り専用。お手元の AI を
  そのまま使う形）。**Claude（Claude Code）** に加えて **Codex（OpenAI Codex）** も選べるように
  なりました（画面上部で切り替え）。Codex は安全のため読み取り専用で動きますが、ツールの絞り込みが
  Claude より緩いため **試験的対応** の位置づけです（常用は Claude をおすすめ）。
- **テキストのみ取得**
  PDF / Excel のファイルは取らず、**項目の入力値（テキスト）だけ**を軽く取り込むモードを追加。
  検索・分析が目的のときに高速・軽量です。

### 改善

- **「現場AIチャットへ」の導線を整理**
  データ／実行から現場 AI チャットへ移動するボタンの表記を **「現場AIチャットへ」** に統一しました。
  また、実行が**進行中（実行中）のときはこのボタンを出さない**ようにしました（結果が出てから相談できます）。
- **配信先を種別ごとに整理**
  送り先を「データベース / データレイク / クラウドDWH / 検索 / アーカイブ」などの種別で
  グループ表示し、目的の送り先を見つけやすくしました。
- **「プラグイン」タブの使い勝手**
  表示を高速化し、各プラグインの「フォルダを開く」「アンインストール」を追加しました。
- **プラグインの更新に気づける**
  アプリに同梱された新しい版がインストール済みより新しいとき、「プラグイン」タブで
  「更新あり」を表示するようにしました（ネット接続なしでも判定）。ワンクリックで最新へ更新できます。
  ※ 例えばアーカイブの送り先に Google Cloud Storage / Azure Blob / ファイルサーバーが
  増えても、更新すれば選べるようになります。
- **状態バッジから対処へ**
  「版が変わった」「出所が確認できない」などのバッジを押すと、対処方法やマニュアルへ案内します。
- **使い方ガイドを刷新**
  専門用語を整理し、よりわかりやすい説明に。仕組みの詳細は各ページ末尾の「技術メモ」にまとめています。
- **接続先の切り替え・復活を安全に・気づける形で**
  i-Reporter の接続先が **切り替わった**ときに気づけるよう、画面でお知らせ＋ワンクリックで各フローを

  データ確認などの詳細パネル（右からせり出すドロワー）の横幅を大きくし、情報を見やすくしました。

### 修正

- **配信済みなのに「データが無い」と表示される問題を修正（重要）**
  i-Reporter の接続先によっては、配信が成功しているのに「現場蓄積データ」や「現場 AI チャット」で
  データセットが 1 件も出ない（AI が「データがありません」と答える）ことがある問題を修正しました。
  接続先の識別方法のずれが原因で、本番のサーバー設定でほぼ常に起きていました。**過去に配信済みのデータも、
  再配信なしでそのまま見えるようになります。**
- **BigQuery への配信を安定化**
  すでに用意済みのデータセットに送るとき、配信が失敗してしまうことがある問題を修正しました。
- **接続先の切り替え時にログインが保存できない問題を修正**
  接続先を切り替えるとき、「接続テストは成功するのに『検証して保存』で認証に失敗する」現象を
  解消しました。テストと保存が同じ確認を行うようになり、入力した接続先・ユーザー・パスワードを
  そのまま検証して保存します（誤った情報を入れても、動作中の接続設定は壊しません）。
- **Windows で現場 AI チャット（Codex）がデータを読めない問題を修正**
  Windows で Codex を選んだとき、AI が現場データの読み取りツールを使えない／呼び出しが取り消される
  問題を解消しました（読み取り専用は維持）。
- **Mac でアプリから i-repo CLI が見つからない問題を修正**
  i-repo CLI を入れているのに「CLI が見つかりません」と表示されることがある問題を解消しました。
  アプリのアイコンから起動した場合でも、nvm / pnpm / Homebrew などで入れた CLI を正しく見つけます。
- **接続先が未設定のときのダッシュボード表示を整理**
  接続先（i-Reporter）が未設定のときは、概要画面で連携状況の数字を出さずに「接続先を設定する」案内を
  表示するようにしました（これまでの実行履歴は消えません・表示の出し分けのみ）。
- **Mac でアプリから Node.js が「未検出」になる問題を修正**
  Node.js（22.5 以上）を入れているのに、アプリのアイコンから起動すると現場 AI チャット／エージェント
  接続の準備状況で「未検出」と表示されることがある問題を解消しました（i-repo CLI の検出と同じ仕組みに
  そろえ、nvm / pnpm / Homebrew などで入れた Node.js も見つけます）。
- **長い帳票名のアーカイブ取得が失敗する問題を修正**
  帳票名が長い（日本語などを含む）と、PDF/Excel の取得が「name too long」エラーで失敗することがある
  問題を解消しました。あわせて、アーカイブ内のファイル名を **「帳票ID_帳票名」で統一**し、一覧で
  見つけやすくしました。
- **破壊的操作の確認で同じ項目が重複表示される問題を修正**
  アーカイブのクリーンアップ確認ダイアログで、同じ注意（クリーンアップ）が複数回表示される問題を
  解消しました。

### 配布

- **macOS アプリを署名・公証（Gatekeeper 警告が出ない）**
  macOS 版を **CIMTOPS の Developer ID で署名し、Apple の公証（notarization）**を通すようにしました。
  初回起動時の「**悪意のあるソフトウェアか確認できません**」という Gatekeeper 警告が**出ずにそのまま開けます**。

  初回起動時に SmartScreen 警告が出ることがありますが、[安全な開き方](manual-setup.html#install-warning)で起動できます。
